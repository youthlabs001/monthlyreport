# 매출 대시보드

고객이 로그인하여 본인 회사의 매출 정보 및 추이를 분석한 데이터를 시각화하여 볼 수 있는 웹 애플리케이션입니다.

## 📊 주요 기능

### 1. 사용자 인증
- 로그인 및 회원가입 시스템
- 데모 계정 제공
- 세션 관리

### 2. 매출 대시보드
- **실시간 통계 카드**
  - 이번 달 매출
  - 전월 매출 및 증감률
  - 평균 거래액
  - 목표 달성률

### 3. 데이터 시각화
- **월별 매출 추이**: 현재년도와 전년도 비교 라인 차트
- **카테고리별 매출 비중**: 도넛 차트로 카테고리별 매출 분포 표시
- **주간 매출 추이**: 요일별 매출을 막대 차트로 시각화
- **분기별 성장률**: 성장률 추이를 라인 차트로 표시

### 4. 거래 내역
- 최근 거래 내역 테이블
- 날짜, 거래처, 카테고리, 금액, 상태 표시
- 상태별 색상 구분 (완료/대기/취소)
- 업로드된 데이터 자동 반영

### 5. 관리자 기능
- **엑셀 데이터 업로드**
  - 엑셀 양식 다운로드
  - 드래그 앤 드롭 파일 업로드
  - 데이터 검증 및 오류 표시
  - 사용자별 매출 데이터 일괄 등록
- **사용자 관리**: 사용자 목록 조회 및 관리
- **회사 관리**: 회사 정보 조회 및 관리
- **데이터 관리**: 백업, 내보내기, 동기화

### 6. PDF 내보내기
- 월별 매출 리포트를 PDF로 다운로드
- 주요 지표, 카테고리별 매출, 거래 내역 포함
- A4 사이즈 인쇄 최적화

```
monthly report/

├── dashboard.html          # 매출 대시보드
├── admin.html              # 관리자 페이지
├── css/
│   ├── style.css          # 공통 스타일
│   ├── dashboard-chart.css # 대시보드 스타일
│   └── admin.css          # 관리자 페이지 스타일
├── js/
│   ├── config.js          # 설정 및 데모 데이터
│   ├── auth.js            # 로그인 기능

```

### 실행 방법

1. **로컬 서버 실행**
   ```bash
   # Python 3를 사용하는 경우
   python -m http.server 8000
   
   # 또는 Python 2를 사용하는 경우
   python -m SimpleHTTPServer 8000
   
   # 또는 Node.js http-server를 사용하는 경우
   npx http-server
   ```

2. **브라우저에서 접속**
   ```
   http://localhost:8000
   ```

3. **데모 계정으로 로그인**
   - **회사A (테크노바 주식회사)**
     - 이메일: `demo1@company.com`
     - 비밀번호: `demo123`
   
   - **회사B (미래산업 코퍼레이션)**
     - 이메일: `demo2@company.com`
     - 비밀번호: `demo123`

## 📤 관리자 - 매출 데이터 업로드

### 엑셀 업로드 사용 방법

1. **관리자 페이지 접속**
   - 로그인 후 좌측 메뉴에서 "관리자" 클릭
   - "데이터 관리" 탭 선택

2. **엑셀 양식 다운로드**
   - "📥 엑셀 양식 다운로드" 버튼 클릭
   - `매출데이터_업로드양식.xlsx` 다운로드

3. **데이터 입력**
   - 다운로드한 양식에 데이터 입력
   - **필수 항목**: 날짜, 거래처, 카테고리, 금액, 상태
   - **날짜 형식**: YYYY-MM-DD (예: 2026-02-09)
   - **금액**: 숫자만 입력 (예: 15000000)
   - **상태**: completed, pending, cancelled 중 하나

4. **파일 업로드**
   - 대상 사용자 선택 (demo1@company.com 또는 demo2@company.com)
   - 작성한 엑셀 파일을 드래그 앤 드롭 또는 클릭하여 선택
   - "⬆️ 데이터 업로드" 버튼 클릭

5. **결과 확인**
   - 업로드 성공/실패 건수 표시
   - 오류 발생 시 상세 내용 확인
   - 해당 사용자의 대시보드에서 데이터 확인

### 엑셀 양식 예시

| 날짜 | 거래처 | 카테고리 | 금액 | 상태 | 비고 |
|------|--------|----------|------|------|------|
| 2026-02-01 | (주)샘플거래처 | 제품 판매 | 15000000 | completed | |
| 2026-02-02 | 테스트기업 | 서비스 | 8500000 | completed | |
| 2026-02-03 | 예시회사 | 구독 | 2500000 | pending | 확인 필요 |

## 🔐 보안 고려사항

현재 버전은 **데모 목적**으로 제작되었으며, 실제 운영 환경에서는 다음 사항을 추가해야 합니다:

1. **서버 사이드 인증**
   - JWT 토큰 또는 세션 기반 인증
   - 비밀번호 해싱 (bcrypt, argon2)
   
2. **HTTPS 사용**
   - SSL/TLS 인증서 적용
   
3. **입력 검증**
   - 서버 사이드 입력 검증
   - SQL Injection 방지
   - XSS 방지
   
4. **CSRF 보호**
   - CSRF 토큰 구현

## 📊 데이터베이스 연동

`database/schema.sql` 파일에는 다음 테이블이 정의되어 있습니다:

- `companies` - 회사 정보

- `monthly_goals` - 월간 목표
- `revenue_records` - 매출 데이터
